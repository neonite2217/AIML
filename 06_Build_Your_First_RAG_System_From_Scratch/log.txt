PROJECT OVERVIEW:
-----------------
This project demonstrates a Retrieval-Augmented Generation (RAG) system built from scratch in Python. The system allows Large Language Models (LLMs) to answer questions based on a private knowledge base.

PROJECT STRUCTURE:
------------------
1. guide.txt         - Project description and workflow documentation
2. requirements.txt  - Python dependencies (faiss-cpu, sentence-transformers, transformers, torch)
3. my_knowledge.txt  - Sample knowledge base with information about Gemini model
4. rag_from_scratch.py - Main implementation file

RAG PIPELINE ARCHITECTURE:
---------------------------
The system implements a 6-step RAG pipeline:

Step 1: Text Chunking
- Loads my_knowledge.txt (creates dummy data if not exists)
- Splits text into smaller, non-overlapping chunks (line-based)
- Each line becomes a separate chunk

Step 2: Embedding Creation
- Uses SentenceTransformer with 'all-MiniLM-L6-v2' model
- Converts text chunks into 384-dimensional numerical vectors
- Enables semantic similarity search

Step 3: Vector Store (FAISS)
- Creates FAISS IndexFlatL2 index (L2 distance metric)
- Stores all chunk embeddings in the index
- Enables efficient similarity search at scale

Step 4: Query Embedding & Retrieval
- Embeds user query using the same sentence transformer
- Searches FAISS index for top-k most similar chunks
- Returns indices and distances of closest matches

Step 5: Prompt Template Construction
- Combines retrieved chunks with original query
- Creates structured prompt for the LLM
- Format: "Context: {retrieved_chunk}\nQuestion: {query}\nAnswer:"

Step 6: Answer Generation
- Uses Hugging Face question-answering pipeline
- Model: distilbert-base-cased-distilled-squad
- Generates answer grounded in retrieved context

KEY TECHNOLOGIES:
-----------------
- FAISS (Facebook AI Similarity Search) - Vector indexing and retrieval
- Sentence Transformers - Text embedding (all-MiniLM-L6-v2)
- Transformers - Question answering pipeline (DistilBERT)
- PyTorch - Backend for neural network computations
