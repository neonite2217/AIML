## 12. Data Cleaning Pipeline using Pandas (Python)

### Description
A reusable cleaning pipeline standardizes data types, handles missing values, cleans text, and manages outliers to make data analysis and modeling reliable and reproducible. This project focuses on building a robust and reusable data cleaning function using the powerful Pandas library.

### Prerequisites
Pandas data cleaning, missing value strategies, type casting, basic stats.

### Resources
`loan_prediction.csv` dataset.

### Project Workflow
The project starts with data inspection. You will load the `loan_prediction.csv` dataset into a Pandas DataFrame and perform an initial exploration to understand its schema, identify columns with missing values, and get a sense of the data quality.

Based on this inspection, you will implement a series of cleaning steps. This includes imputing missing values (using the median for numerical columns and the mode for categorical ones), correcting column data types, and standardizing string data by trimming whitespace and converting to a consistent case. You will also implement a method to cap outliers using a percentile-based approach.

The final and most important step is to encapsulate all these cleaning operations into a single, reusable Python function or class. This function will take a raw DataFrame as input and return a cleaned DataFrame, making it easy to apply the same cleaning logic to new datasets in the future.

### Lab steps
1. Inspect schema and missing values.
2. Impute categorical with mode; numeric with median.
3. Fix column data types.
4. Standardize strings (trim whitespace, lowercase).
5. Cap outliers using percentiles (e.g., 99th).
6. Wrap all steps into a function for reuse.

### How you may enhance this project
- **Automated Memory Optimization:** Large datasets can consume a lot of memory. Enhance your pipeline by adding a step that automatically infers the optimal data types for each column. For example, a column of integers that only ranges from 0 to 100 doesn't need a 64-bit integer type. Using a library like `dtype_diet` or writing your own function to downcast numerical types can dramatically reduce memory usage.
- **Comprehensive Logging and Reporting:** Make your pipeline more transparent by adding a robust logging mechanism. At each step, log the operation being performed, the columns being affected, and the number of rows changed. At the end, generate a summary report that quantifies the impact of the cleaning process, such as the total number of missing values imputed and outliers capped.
- **Interactive Data Cleaning Tool:** For a great portfolio piece, build a simple interactive tool using Streamlit or Dash. This tool would allow a user to upload their own CSV file, select the cleaning operations they want to apply from a series of checkboxes and dropdowns, and then see a preview of the cleaned data and download the result.
- **Advanced Outlier Detection Methods:** Capping outliers based on percentiles is simple, but not always the most effective method. Implement more robust outlier detection algorithms like using the Z-score for normally distributed data, or more advanced, model-based techniques like DBSCAN or Isolation Forest, which can detect outliers in a multi-dimensional space.
- **Standardize Categorical Data with Fuzzy Matching:** Categorical columns often suffer from inconsistent entries (e.g., "New York", "NYC", "N.Y."). Implement a fuzzy matching algorithm (e.g., using the `fuzzywuzzy` library) to group these variations and standardize them into a single, consistent category. This is a common and very practical data cleaning task.
